# Interview Prep Phases

## Phase 1 - pattern recognition first

For 2-3 days:
- do not code immediately
- read a problem
- say:
  - input shape
  - likely pattern
  - why not other patterns
  - brute force
  - optimized idea

Goal:
- train the diagnosis muscle

## Phase 2 - coding templates

For the next 4-5 days:
- take only easy/medium problems from those 8 patterns
- code from memory
- after each problem, write:
  - pattern
  - key clue
  - bug made
  - better implementation habit

## Phase 3 - interview simulation

Then:
- 35 minutes per medium problem
- speak out loud:
  - brute force
  - optimized approach
  - complexity
  - edge cases

## Best Problem Order For You

Do them in this order, not randomly:
- Hash map
- Two pointers
- Sliding window
- Binary search
- Linked list
- Stack
- Tree BFS/DFS
- Heap

Why this order?
- low implementation complexity first
- pattern clues become clearer
- confidence builds faster
- trees/heaps feel easier after arrays/strings are stable
