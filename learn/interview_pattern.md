# Interview Patterns

For your case, start with these 8 interview patterns only:

## 1. Hash Map / Set
**Signals:**
- lookup fast
- duplicates
- frequency count
- seen before
- pair sum / complement

## 2. Two Pointers
**Signals:**
- sorted array
- pair/triplet
- opposite ends
- remove duplicates
- palindrome

## 3. Sliding Window
**Signals:**
- subarray / substring
- contiguous segment
- longest / shortest window
- "at most k"
- frequency inside a window

## 4. Binary Search
**Signals:**
- sorted data
- first/last position
- search space is monotonic
- "minimum feasible" / "maximum possible"

## 5. Stack
**Signals:**
- matching brackets
- previous/next greater
- undo/backtrack feel
- expression parsing
- monotonic behavior

## 6. Linked List
**Signals:**
- reverse
- cycle
- merge
- middle node
- pointer rewiring

## 7. Tree / DFS / BFS
**Signals:**
- hierarchy
- level order
- path
- subtree
- depth / traversal / nearest

## 8. Heap / Priority Queue
**Signals:**
- top K
- kth largest/smallest
- streaming data
- always need min/max quickly
